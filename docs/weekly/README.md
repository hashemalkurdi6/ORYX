# Weekly review

Every Friday, write a summary at `docs/weekly/YYYY-WW-week-of-Mon-DD.md` (e.g. `2026-17-week-of-Apr-20.md`).

## Template

```markdown
# Week of [Date range]

## Shipped
- [What got merged this week, with commit hashes]

## Cut or deferred
- [What was planned but moved to v1.1]

## Lessons
- [What worked, what didn't]

## Next week
- [Priorities]

## Honest assessment
[On track / behind / ahead — and why]
```

## Tips

- "Honest assessment" is the only section that matters when you re-read these in 8 weeks. Be blunt — past-you is the audience.
- "Cut or deferred" is the source of truth for the v1.1 backlog. Anything that lives here long enough should graduate to a tracked item.
- If a week has nothing in "Shipped," that's the whole story — don't pad it.
