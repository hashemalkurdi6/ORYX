# Daily notes

One file per day, named `YYYY-MM-DD.md`.

## Template

```markdown
# YYYY-MM-DD

## Working on
- [What I'm tackling today]

## Open questions
- [What I'm stuck on]

## Done
- [Completed items, with commit hashes if relevant]

## Blockers
- [Anything preventing progress]

## Tomorrow
- [Next priorities]
```

## Tips

- Don't write a daily note "for the future" — write at the start of the day, edit through it, finalise at the end.
- "Done" should link commits with short SHAs (e.g. `1c938d1 — activity audit fixes`).
- "Open questions" is the most underrated section. If you find yourself stuck twice on the same thing, it deserves a decision in `docs/decisions/`.
